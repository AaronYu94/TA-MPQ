TA-MPQ: Structured Exact-Budget Mixed Precision for Task-Aware LLM Quantization

Aaron Yu  ·  Letian Zhang  ·  Tim Gao
Carnegie Mellon University  ·  15-442/15-642: Machine Learning Systems
Key Takeaway
Our Structured Exact-Budget Coarse-to-Fine Mixed-Precision Search consistently recovers quality lost by uniform INT4 — under the same memory budget.
Motivation & Problem
1

Running LLMs under strict memory budgets is critical for local inference and multi-LLM orchestration. Uniform quantization treats all layers identically — but components vary widely in task sensitivity.

  • Assigning equal precision wastes budget on low-sensitivity layers
  • Different benchmarks value different groups — one policy need not fit every workload family
  • Goal: reallocate precision across groups without increasing total budget

We target the same raw weight footprint as uniform INT4 and reallocate precision across groups instead of increasing total budget.

Why Not Evolutionary Search?
3

As originally proposed, we tried an evolutionary algorithm. In practice, crossover made it too difficult to find policies that both improved performance and stayed under budget within limited compute.

We do not argue evolutionary search is universally ineffective — it was unreliable under strict exact-budget constraints and limited compute in our setting.

Core Idea
2

Unlike You Had One Job (fixed 25% INT8 / 75% INT4, no strict budget), we use fixed-ratio recipes only as sensitivity profilers. We rank all linear groups by task sensitivity / true group size, then build an exact-budget policy frontier:

  • High-sensitivity groups → promoted to INT8
  • Low-value large groups → demoted to INT2 to pay for INT8 cost
  • Remaining groups → stay at INT4

Every candidate stays within the exact INT4 raw weight footprint. Left end: more INT8+INT2. Right end: uniform INT4.

Policy Frontier
4a

Each point on the frontier is an exact-budget policy. As we promote more INT8, we must add INT2 to stay within budget — spanning a full spectrum.

Observation: Adjacent policies on the frontier tend to have similar benchmark performance — the space is discrete but behaves like a nearly continuous function. This justifies coarse-to-fine search: a good sector reliably contains a good policy.

The strongest claim is not that mixed precision beats INT8. The defensible claim is:

Limitations & Future Work
6

What We Claim

  • Beats uniform INT4 on HumanEval, BigCodeBench-Hard, and MATH-500
  • Does not consistently beat INT8
  • Current evidence supports workload-family-aware policies — not a single universal policy

Current Limitations

  • Benchmark sizes are modest; small differences should be interpreted carefully
  • Latency is reported for completeness — current comparisons are influenced by the serving backend; token usage and accuracy are more reliable indicators at this stage

Next Steps

  • Larger model families and longer experiments
  • Layer-level vs. component-level granularity comparison
  • Optimize directly for token usage, not only accuracy
  • Controlled cross-task experiments to test true task specificity
