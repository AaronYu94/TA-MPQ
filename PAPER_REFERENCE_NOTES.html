当前路线下的决策区块

这一块只回答现在该吸收什么、接下来该做什么，以及哪些方向先不要扩。

已经对齐当前主线的点

  • You Had One Job：TAQO 和 calibration-vs-policy framing，已经和当前 targeted ablation 方向高度对齐。
  • AWQ：activation-aware saliency 已经和我们现有的 prior / feature 设计兼容。
  • ScaleBITS：progressive sensitivity 和 budgeted allocation framing，最接近当前 uniform8-aware 路线。
  • RAMP：policy search、layer feature 和 reward 设计，最适合继续补强我们现在的搜索器。

下一步最值得吸收的内容

  • 从 ScaleBITS 借 “围绕当前量化参考点估计 sensitivity” 的表述，强化现在 ablation-driven prior 的理论依据。
  • 从 RAMP 借 layer feature 分组方式，补强 surrogate dataset 的结构化特征。
  • 从 You Had One Job 借 TAQ-KL 类信号，作为现有 ablation 和 activation signal 之间的补充。
  • 从 AWQ 借 “只保护极少数真正值钱部分” 这条预算直觉，继续约束 16-bit 分配。

当前不建议优先推进的方向

  • MorphServe 式 runtime adaptation：对未来部署故事有帮助，但不是当前主瓶颈。
  • 直接切 RL：RAMP 很有参考价值，但现在更值得先借 reward 和 feature，而不是整条训练范式换掉。
  • 只靠 GPTQ/AWQ 做结论：它们更适合 baseline / reference，不足以替代 task-aware search 这条主线。

如果把这页当知识库，现在最先看哪几篇

  • 先看 ScaleBITS：它最接近当前预算约束 mixed-precision allocation 的问题定义。
  • 再看 RAMP：它最接近当前搜索器层面的相关工作。
  • 再看 You Had One Job：它最适合补 task-aware signal 和 ablation 设计。
  • AWQ / GPTQ：主要作为 baseline 和信号设计参照。
当前最值得吸收的组合 如果只选一组最适合现在立刻并入 TA-MPQ 的内容，优先是： ScaleBITS 的 progressive sensitivity + RAMP 的 feature / reward design + You Had One Job 的 task-aware ablation framing。

Paper 01 Per-task PTQ Task-aware mixed precision Relevant to current route

You Had One Job: Per-Task Quantization Using LLMs’ Hidden Representations

这篇论文的核心价值在于：它把“不同任务应当保护不同层的精度”这件事， 从动机、信号设计、实验对比和 ablation 结构上都讲完整了。

这篇论文最值得引用的核心立场

  • 常见 PTQ 往往是 task-agnostic，会把 bit 花在对目标任务没那么重要的层上。
  • 应该用目标任务的小规模 calibration prompts 估计 layer importance，再做 mixed-precision allocation。
  • 任务相关的隐藏表示、单层量化掉点、输出分布敏感度，都可以成为 per-layer precision 的依据。

和当前 TA-MPQ 的关系

  • 它和我们当前的方向一致：都在做 task-aware mixed-precision PTQ。
  • 它更偏“先给层打分，再直接分 bit”。
  • 我们更偏“把任务信号喂给 surrogate 和 evolutionary search，再在预算约束下搜 policy”。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Task-aware PTQ framing 直接借 这篇把“任务相关层需要更高精度”说得非常清楚，适合放在动机和 related work 里。 作为我们当前 uniform8-aware 4/8/16 路线的理论背景。
TAQO：单层/单组量化敏感度 直接借 这和我们现在做的 16→8 / 8→4 targeted ablation 本质一致。 继续把真实掉点写进 group value prior，作为下一轮搜索的强监督信号。
TAQ-KL：输出分布敏感度 改造后借 它不需要标签，适合当额外的 label-free signal，尤其适合当前 surrogate 还偏弱的时候。 把层扰动后的 logits KL 作为新特征，接到 surrogate dataset 或 prior 里。
TAQ：hidden representation scoring 改造后借 它把 hidden-state entropy 和 activation stability 合成层分数，这和我们已有的 sensitivity/profile 思路兼容。 作为现有 task sensitivity / representation prior 的补充，而不是替代 closed-loop search。
Calibration vs Policy ablation 直接借 它专门验证“收益到底来自 on-task calibration，还是来自 task-aware allocation policy 本身”。 我们后面也应该把问题拆成：task data 对了，和 policy 真学会了，两者分别贡献多少。

最建议立刻吸收的部分

  • 把 TAQO 当成我们当前 targeted ablation 的文献对齐版本。
  • 把 TAQ-KL 做成新的 label-free layer/group sensitivity signal。
  • 把他们的 ablation framing 用到我们自己的实验结构里，而不只是复述结果。

不建议直接照搬的部分

  • 不要直接把他们的“按分数排序后直接分 bit”替换掉我们现有的 surrogate + evolutionary search。
  • 不要直接照抄他们的 benchmark 组合；他们偏 TriviaQA / CodeMMLU / MMLU-Pro，我们当前主线还是 MATH-500。
  • 不要直接把他们的 layer 级别假设原样搬过来；我们现在主要在做 per_block_component 的 group-level policy。

建议在自己论文里怎么引用它

  • 用它支持 “task-agnostic PTQ 会错配精度预算” 这个动机。
  • 用它支持 “task-specific hidden signals / output sensitivity 可以指导 mixed-precision allocation” 这个方法论。
  • 用它对齐我们现在的 targeted ablation：我们不是凭感觉调层，而是在做 task-level sensitivity estimation。
  • 同时明确区别：他们是 score-then-allocate，我们是 signal-to-surrogate-to-search。
当前最有价值的借法 如果只选一个最适合现在吸收的点，优先借 TAQO： 也就是把“单层/单组降精度后的真实掉点”正式定义成 task-aware sensitivity signal， 再回喂到我们现有的 surrogate 和 search 里。
Paper 02 Survey Mixed-precision taxonomy Terminology / framing reference

Mixed-Precision Quantization for Language Models: Techniques and Prospects

这篇不是提出一个新 task-aware 算法，而是给 mixed-precision LM quantization 做了一张非常系统的地图。它最适合借的不是某个具体分配策略，而是术语、taxonomy、 任务边界和 related-work framing。

这篇论文最有价值的地方

  • 它把 mixed precision 在语言模型中的定义讲清楚了。
  • 它区分了 inter-layer 和 intra-layer allocation。
  • 它把 mixed-precision LM 框架按 MPW / MPW+UPA / MPW+MPA 做了结构化 taxonomy。
  • 它把 weights、activations、KV cache、granularity 和 hardware constraints 放到了同一张图里。

和当前 TA-MPQ 的关系

  • 它不会直接告诉我们哪几个 group 应该上 16-bit。
  • 但它非常适合拿来统一我们现在文档和论文里的语言，避免 mixed-precision 这个词用得太松。
  • 对于当前 TA-MPQ，它更像“地图和术语层”的参考，而不是“搜索信号层”的参考。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Mixed precision 的严格定义 直接借 它明确指出：只有权重低比特、激活保持 FP，或者权重统一低比特而激活统一另一种比特，不应在严格意义上都算 mixed precision。 统一我们在 README / HTML / paper 里的措辞，避免把所有非全精度配置都叫 mixed precision。
Inter-layer vs Intra-layer distinction 直接借 这正好能把我们当前的 per_block_component 说得更清楚：我们不是只在 layer 级别分 bit，而是在更细的 group/component 级别做 allocation。 把 TA-MPQ 明确表述成更接近 intra-layer mixed-precision weight allocation 的框架。
MPW / MPW+UPA / MPW+MPA taxonomy 直接借 它给了一个很好的分类框架，能帮助我们把自己的方法定位到更准确的位置。 当前 TA-MPQ 可以明确定位成 MPW：mixed-precision weights，activations 不作为当前主优化对象。
Granularity taxonomy 直接借 它系统整理了 per-tensor / per-channel / per-group / per-token，这对解释我们为什么选 group-level 很有用。 把当前 group-wise affine quantization + per_block_component allocation 放到这个 taxonomy 里讲清楚。
Framework-level literature map 改造后借 它总结了 GPTQ、AWQ、SmoothQuant、QuaRot、OBS-style 等家族，更适合 related work，而不是直接拿来实现。 用来组织背景章节：我们的方法是 mixed-precision search，不是单纯等价变换或单层重建型 PTQ。
Hardware-aware discussion 改造后借 它提醒 mixed precision 不只是算法问题，还和 runtime、硬件支持、KV cache/activation handling 相关。 后面如果我们补 deployment story，这篇可以作为“为什么只支持某些 bit 配置更现实”的引用来源。

最建议吸收的部分

  • 把 mixed precision 的定义说严谨。
  • 把我们的方法明确放进 MPW 这一类。
  • 把当前搜索单位说明成介于 inter-layer 和 intra-layer 之间、偏 group/component-level 的分配。
  • 用它的 granularity taxonomy 解释为什么当前我们坚持 group-level 而不是 parameter-level。

不建议直接照搬的部分

  • 它是 survey，不提供一个可以直接替代我们搜索系统的新算法。
  • 不要把它当成“怎样训练进化算法”的方法来源；它更像框架地图。
  • 不要因为它覆盖 activations / KV cache，就把当前项目范围无控制地扩展过去。

建议在自己论文里怎么引用它

  • 用它支持 mixed-precision LM quantization 的整体背景和 taxonomy。
  • 用它定义我们当前方法的 precise scope：当前 TA-MPQ 属于 mixed-precision weights，而不是 activation/KV-aware 全栈量化。
  • 用它帮助说明我们的方法和 GPTQ/AWQ/SmoothQuant 这些家族是什么关系。
  • 用它来规范“mixed precision”这个词，避免表述过松。
当前最有价值的借法 这篇最适合借的是 定义、taxonomy、scope framing， 而不是一个具体的 layer scoring rule。它能帮我们把 TA-MPQ 放在 mixed-precision LLM quantization 这张大图里讲清楚。
Paper 03 PTQ baseline Hessian-aware reconstruction Baseline / sensitivity reference

GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers

这篇更适合当 baseline 和方法参照系。它不是 task-aware mixed-precision search，但它把 Hessian-aware、layer-wise PTQ 和 second-order reconstruction 这条线定义得很清楚。

最值得借的点

  • 把 PTQ 写成 layer-wise reconstruction 问题。
  • 用 approximate second-order / Hessian 信息衡量量化误差的重要方向。
  • 把 4-bit 附近的高精度 PTQ 做成强 baseline，而不是只用简单 RTN。

对 TA-MPQ 的作用

  • 它更适合做 baseline 和 sensitivity metric reference，而不是直接替代我们的 search。
  • 可以支持“纯 layer-wise reconstruction 不等于 task-aware allocation”这层对比。
  • 也适合引用到 related work 里，说明我们不是从零开始做 mixed-precision PTQ。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Hessian-aware reconstruction framing 直接借 它给了 PTQ 一个非常清楚的优化表述，适合作为我们 mixed-precision search 的低层量化参考系。 在 related work 里明确：我们不是只做 layer-wise reconstruction，而是在其上再做 task-aware allocation。
Second-order sensitivity intuition 改造后借 虽然 GPTQ 自己不做 task-aware search，但它说明“不同方向的误差代价不同”。 可以把 Hessian/Fisher 近似当作额外 group feature，补进 surrogate dataset。
Strong PTQ baseline role 直接借 它是你们 paper 里最自然的 canonical PTQ baseline 之一。 后面如果补更完整 baseline，对标对象里应当明确出现 GPTQ 家族。
最合适的引用姿势 把 GPTQ 放在 “Hessian-aware PTQ / reconstruction-based PTQ baseline” 这一类， 用来衬托我们的方法是 task-aware signal + surrogate + search， 而不是只做统一的 layer-wise reconstruction。
Paper 04 PTQ baseline Activation-aware saliency Signal design reference

AWQ: Activation-Aware Weight Quantization for On-Device LLM Compression and Acceleration

这篇最值得借的是“activation-aware saliency”这套信号设计， 不是它最后的实现形式。它说明 activation statistics 本身就能提供很强的量化重要性线索。

最值得借的点

  • 不是所有权重同样重要，应该看 activation distribution 来识别 salient channels。
  • 它明确反对硬件不友好的随意 mixed precision，转而用 scaling 去保护 salient channels。
  • 不依赖 backprop 或 reconstruction，校准成本低，泛化也更稳。

对 TA-MPQ 的作用

  • 非常适合支持我们“activation-based task signal”这条线。
  • 也适合用作 uniform / non-search baseline 的代表。
  • 但它不是直接告诉我们怎么做 4/8/16 search 的论文。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Activation-aware saliency 直接借 这和我们现在的 task sensitivity / ablation prior 很兼容，都是在找“哪些 group 更值得保精度”。 继续把 activation statistics 作为 surrogate feature 和 group prior 的一部分。
Protect a small salient subset 改造后借 AWQ 的核心不是“全部升 bit”，而是“只保护最关键的小部分”。 这和我们现在在找“哪些 group 值得 16-bit”是同一个预算逻辑。
Hardware-aware caution 直接借 它明确提醒 mixed precision 不能只看精度，还要看 runtime 友好性。 可以用来支持我们为什么坚持 group-level、budgeted、hardware-aware 搜索，而不是任意细粒度保护。
最合适的引用姿势 把 AWQ 放在 “activation-aware but not task-conditioned” 这一类： 它说明 activation 信号很有价值，而我们进一步把这种信号和任务分数、closed-loop search 绑在一起。
Paper 05 Serving / systems Runtime adaptation Secondary relevance

MorphServe: Efficient and Workload-Aware LLM Serving via Runtime Quantized Layer Swapping and KV Cache Resizing

这篇不直接解决你们当前的 layer-wise search 问题，但它对“运行时按压力切换量化层”和 “sensitivity-based profiling”这两点很有启发。

最值得借的点

  • 它把“哪些层影响小，适合在高压时切成量化版本”做成了 sensitivity-based profiling。
  • 它的 layer swapping 强调状态保持和低 runtime overhead。
  • 它明确把精度选择放到 workload-aware serving 语境下，而不只是离线压缩。

对 TA-MPQ 的作用

  • 不是当前核心算法参考。
  • 更适合当“如果以后要讲 deployment / serving story，可以往哪扩”的参考。
  • 也能支持我们把 ablation profile 解读成“impactful vs less-impactful groups”。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Sensitivity-based profiling for layer swapping 改造后借 它和我们现在的 targeted ablation 很贴，只是目标从“runtime layer swapping”换成了“static allocation search”。 把 ablation 掉点进一步解释成 “这些 groups 是 high-impact / low-impact”。
Accuracy-latency trade-off framing 改造后借 如果后面要把结果写到更系统的部署故事里，这篇的 framing 会很好用。 作为未来扩展方向，不建议现在直接改主线去做 runtime adaptation。
最合适的引用姿势 这篇更像 deployment-side reference。它可以支持 “layer impact profiling 是有系统价值的”， 但不建议把它当成当前 TA-MPQ 训练进化算法的直接方法来源。
Paper 06 Directly relevant Bitwidth search Hardware-aligned MPQ

ScaleBITS: Scalable Bitwidth Search for Hardware-Aligned Mixed-Precision LLMs

这篇和当前 TA-MPQ 的问题最接近。它把 mixed-precision allocation 明确写成预算约束下的资源分配问题， 而且非常强调 hardware-aligned partitioning 和可扩展的 bitwidth search。

这篇为什么特别重要

  • 它直接讨论了“预算约束下，怎么做 automated fine-grained bitwidth allocation”。
  • 它指出经典 greedy / RL / evolutionary 在超大搜索空间下会很贵，所以必须用可扩展近似。
  • 它还把 hardware-friendly partitioning 放在了方法中心，而不是最后补 runtime 解释。

和 TA-MPQ 的关系

  • 它是你们当前 uniform8-aware 路线最值得正面对照的一篇。
  • 它和我们一样关注预算、bit allocation、hardware alignment。
  • 不同点在于：它更像 sensitivity-guided scalable allocation；我们是 surrogate + evolutionary closed loop。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Resource-allocation framing 直接借 它把 mixed-precision allocation 写成 knapsack-style discrete resource allocation，这和我们当前预算约束目标高度一致。 可以直接拿来强化我们 objective 的理论表述，而不只是工程搜索。
Progressive-quantization sensitivity 直接借 它强调 sensitivity 应该围绕“当前已量化模型”来估计，而不是只看 full-precision 点。 这和我们现在在 current candidate 上做 targeted ablation 非常一致，可以直接对齐成理论依据。
Hardware-aligned block partitioning 改造后借 它说明 mixed precision 的 granularity 不只是统计问题，也是 kernel/partition 问题。 我们现在的 per_block_component 可以借它来解释“为什么要选硬件友好的 group-level 单位”。
Scalable approximation to greedy search 改造后借 它很适合当我们当前 evolutionary search 的对照思路：不是所有搜索都要靠 GA/RL。 后面可以把它当 alternative allocator baseline，或者至少借它的分析来解释搜索开销问题。
最合适的引用姿势 如果要给当前 TA-MPQ 找一篇“问题最像”的参考，优先引用这篇。 它最能支持我们现在这条线：预算约束、group-level allocation、search scalability、 以及“量化后的参考点上再估计 sensitivity”。
Paper 07 Directly relevant RL search Policy-based MPQ

RAMP: Reinforcement Adaptive Mixed-Precision Quantization for Efficient On-Device LLM Inference

这篇最像“另一种 search engine”。它把 bit allocation 写成 sequential decision-making， 用 SAC 学 per-layer bit policy，是当前 TA-MPQ 搜索框架最直接的相关工作之一。

这篇为什么重要

  • 它直接反对 uniform bit-width，认为 layer sensitivity 有系统差异。
  • 它不是 heuristic 分配，而是显式学一个 bit-allocation policy。
  • 它还强调 policy transfer，这和我们现在希望 search 不只会记住单次结果很接近。

和 TA-MPQ 的关系

  • 这篇和我们一样在做 mixed-precision policy search。
  • 它用 RL；我们现在用 surrogate-guided evolutionary closed loop。
  • 它非常适合当“搜索策略层”的 related work 和对照框架。

可以直接借鉴的方法点

方法点 借鉴强度 为什么适合当前项目 在 TA-MPQ 里怎么落地
Sequential decision framing 直接借 它把 bit allocation 视为序列决策，这和我们“candidate 不是静态公式，而是搜索结果”非常一致。 可以直接增强我们对进化算法角色的表述：它是 policy search，而不是手工打分器。
Layer feature embedding design 直接借 它用 11 维 layer embedding 喂给 policy，这和我们当前 surrogate feature engineering 很接近。 可以直接参考它的 activation / weight / structural / contextual feature 分组方式，补强当前 dataset 特征。
Quality-prioritized reward 直接借 它明确把质量放在首位、budget 当柔性约束，这和我们当前围绕 uniform8 的 objective 很贴。 可用于继续改进当前 search objective / acquisition，而不一定非得改成 RL。
Policy transfer hypothesis 改造后借 它提出“量化敏感度主要是 architectural”这一假设，这能帮助我们思考当前方法能否跨模型泛化。 先作为 discussion / future work，不建议现在就把主线切成跨模型 transfer 实验。
最合适的引用姿势 这篇最适合拿来对照“搜索策略”层： 他们用 RL 学 bit policy，我们用 surrogate-guided evolutionary search。 两者都在解决同一个核心问题：怎样把 bit budget 分给真正值钱的层。
