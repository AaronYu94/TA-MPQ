Current Results, Consolidated 当前结果总览

This page summarizes the current state of the TA-MPQ project as of April 11, 2026. The mainline is now Qwen3.5-27B compressed toward native 9B budget, with the earlier 9B vs 4B experiment retained as a low-cost debugging track rather than the primary research claim. Mainline Framing 主线问题定义

The project now asks whether a task-aware mixed-precision compressed 27B can remain competitive against a native 9B under roughly the same deployment budget. 当前项目的核心问题是：在大致相同的部署预算下， 一个任务感知、混合精度压缩后的 27B， 能否保持对原生 9B 的竞争力。

Mixed-Precision Feasibility 混合精度可行性

A dry-run feasibility path was built to inspect Qwen3.5-27B layer structure and estimate budget under a simple mixed-bit policy before real PTQ export was available. It now serves as the reference layer-statistics stage for the full pipeline. 在真实 PTQ 导出路径尚未打通时，我们先建立了一条 dry-run feasibility 路径， 用于统计 Qwen3.5-27B 的层结构，并在简单 mixed-bit policy 下估算预算。 现在它则继续作为完整 pipeline 里的 layer statistics 参考阶段。

Real PTQ Validation 真实 PTQ 验证

The project is no longer blocked before full PTQ validation. Runnable quantized 27B artifacts have now been exported and evaluated on both GSM8K and MATH-500, and the surrogate-guided search path is now online. The important update is that the latest multi-iteration closed-loop search has now produced a stronger winner: on the larger limit=100 comparison, the best closed-loop task-aware candidate reaches 40%, slightly above uniform 4-bit at 39%. 项目已经不再卡在 full PTQ validation 之前。现在已经成功导出并评测了 多个可运行的 quantized 27B artifact，不仅跑通了 GSM8K，也跑通了 MATH-500； 现在 surrogate-guided search 这条线也已经接通了。最新的重要更新是： 多轮 closed-loop search 已经产生了更强的新赢家；在更大的 limit=100 对比里，最强 closed-loop task-aware 候选达到 40%，略高于 uniform 4-bit 的 39%。

Surrogate-Guided Search Surrogate 引导搜索

The proposal-style middle loop is now implemented end-to-end and wrapped into a multi-iteration closed loop: executed policies are converted into features, an XGBoost ensemble surrogate is trained, novel candidates are searched automatically, and the best new records can be promoted directly into larger-sample evaluation. proposal 中间那段方法线现在已经端到端接通，并且被包装成了多轮 closed loop： 真实执行过的 policy 会先变成特征，再训练一个 XGBoost ensemble surrogate， 自动搜索新的候选，然后把新的 best record 直接推进到更大样本评测。

Executed dataset 真实执行数据集

The closed-loop manifest now contains 13 real MATH-500 records. The latest v3 loop trained on 12 examples with 45 features before proposing its next candidate. 当前 closed-loop manifest 里已经有 13 条真实执行过的 MATH-500 记录。最新的 v3 loop 在提出下一轮 candidate 之前， 已经用 12 个样本和 45 个特征完成训练。

Current Open Question 当前开放问题

The remaining gap is no longer basic PTQ execution. The question has become empirical: can task-aware mixed precision hold a win over a strong uniform 4-bit baseline consistently, not just narrowly once? The latest MATH-500 limit=100 comparison now says yes, but only by 1/100. 当前剩下的缺口已经不在基本 PTQ 执行本身，而变成了一个经验问题： task-aware mixed precision 能不能不只是赢原生 9B， 还要对一个很强的 uniform 4-bit baseline 保持稳定优势？ 最新的 MATH-500 limit=100 对比给出的答案现在变成了： 这次赢了，但只赢了 1/100。
